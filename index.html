HTML

Hyper Text Markup Language (HTML) is used to create the structure of a webpage. It is made up of elements which work together to provide content and the basic framework for all pages on the web.

The following element types are commonly used to add content to pages:

  • Text elements
  • Image elements
  • Header elements
  • List elements

Here are some common element types that are used to divide and describe a page:

  • Meta-related elements
  • Content division elements
  • Semantic elements

In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage. When possible, it is important to use semantic HTML elements, because they make your HTML easier to read by other developers and the browser. Semantic HTML also plays an important role in making sure your webpage is accessible to the widest variety of users, including those who use assistive technologies such as screen readers.

Each HTML element performs a unique role.

  • The <!DOCTYPE html> declaration is letting the browser know to expect HTML on the page.
  • The <html> opening tag represents the root, or start, of our HTML document.

An HTML attribute defines additional information about an element. For example, within the <html> tag is an attribute of `lang="en"`, which tells the browser the language for the webpage.

Head

The <head> element contains the metadata for our web application. Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance.

Inside the <head> element, we can use the following metadata tags:

  • The <title> element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The <style> element defines the CSS styles associated with a website.
  • The <base> element defines the base URL for a webpage.
  • The <link> element connects an external resource to the HTML doc.
  • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport. Related resource: https://www.w3schools.com/tags/tag_meta.asp
  • The <script> element embeds client-side scripts in the HTML document.

Body

The <body> element defines the body of the document, which contains the content that is visible to the user. All of the content that we want to display to users must be inside the <body> element.

  • The <h1>,<h2>,<h3>,<h4>,<h5>, and <h6> elements represent the level of heading a given text block represents. Headings are exactly what they sound like: larger or more prominent titles and subtitles to organize a page. When using headings, it is important that we use heading tags in order from 1 to 6 to make the page accessible to users who access the page using screen readers or other devices. In addition, the biggest heading, <h1>, should only appear once on each page.
  • The <p> element represents a paragraph or block of text.
  • The <ul>, <ol>, and <li> elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The <img> element contains information about images that are displayed on the webpage. The <img> element can contain various attributes; the `src` attribute is the most important because it defines the location of the image. The `alt` attribute contains a text string that describes the appearance and functionality of the image. The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website.
  • The <br> element creates a line of empty space, or a line break, between two blocks of content.
  • The <a> element (which stands for anchor) creates links to the same webpage or other webpages. The `href` attribute points to the URL for the link.
  • <section> elements are semantic elements that define a section in a document that contains thematic content.
  • The <footer> appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

Example Workflow

Here is an example Git workflow for reference.

We must begin by creating a feature branch, then add changes to the project, and finally merge the feature branch back into main.

  1. In the command line, navigate to the project directory, if you're not already there.
  2. Type git status to check which branch you're currently in. You should be in the 'main' branch. If not, switch to the 'main' branch by entering the following Git command: git checkout main.
  3. In the command line, pull down the latest 'main' branch by entering git pull origin main.
  4. From the 'main' branch, use the following Git command to create a new branch named 'feature/add-html': git checkout -b feature/add-html. You should see the following message in your terminal: Switched to a new branch 'feature/add-html.
  5. Let's confirm we're on the new branch by entering the following command into the terminal: git status.

Now changes can be made to the project.

The following are the steps taken after we have completed changes to a project. A feature branch was created named 'feature/add-html', we made changes to the project while on this branch, now we are ready to merge those changes into the main branch.

  1. Make sure we are on the 'feature/add-html' branch from our command line. We have to add the changed files in our working directory to the staging area of Git. Use the following command in your terminal to stage your changes: git add -A. We use the 'add -A' command to commit all files in the working branch.
  2. Now that all of our changes have been staged, we can commit them to our repository. Use the following command in your terminal to commit the changes: git commit -m "added code to HTML file". We use the -m flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory.
  3. Next, let's make sure that our current branch is up to date before opening a pull request. We need to sync our local branch with the remote branch in GitHub. This is especially important in a team environment in case a team member has added a new feature to the project. The process of syncing and merging code into the GitHub repo is how the project gets continually updated with more features and content from different team members. Use the following command in your terminal to pull in the base branch: git pull origin main. It is a good habit to always pull the base branch into your feature branch before pushing up any code.
  4. Let's push up all of our changes by using the following command: git push origin feature/add-html.
  5. Now navigate to the repository on GitHub. We need to merge our feature branch into the base branch so that in a real work situation other team members can update their feature branches with an updated version of the project. Once you are in the repository, navigate to the 'feature/add-html' branch using the navigation buttons tab.
  6. Click the "Pull Requests" tab at the top.
  7. Once on the pull requests page, find and click the "New pull request" button.
  8. Now we need to compare the two branches that we want to merge. The base branch that we want to merge into should come first. In our case, this is the 'main' branch. We want to merge our 'feature/add-html' branch, so that is our compare branch.
  9. Once on the pull request form, give your PR a title and a short description of the changes.
  10. Click the "Create pull request" button.
  11. It is time to merge our new feature into our main codebase! Click the "Merge pull request" button and confirm.
  12. Navigate back to the 'main' branch in GitHub, and you should see that your changes are now there.
  13. Open up our terminal and update our local repository with the new merged-in feature. Use the following commands in your terminal: git checkout main and then: git pull origin main.
  14. Let's make sure that we close our HTML issue on GitHub since we have merged in the feature associated with it. Navigate to the "Issues" tab on GitHub.
  15. Click the "HTML" issue and then click the "Close issue" button.
